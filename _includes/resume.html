Software Documentation

Although preventing user frustration (through good software design, UX, development, and testing) is the best mode of action, my primary expertise is in developing software documentation to help bridge knowledge gaps, provide tips, and provide procedural steps for users.

I write for a variety of audiences, including application developers, system administrators, and non-technical end users.

My deliverables include: user interface (UI) strings, error messages, tutorials, scenarios, technotes, and other software documentation. I enjoy writing useful material that is easy to use and understand. I author information with Darwin Information Typing Architecture (DITA) in Arbortext Editor and Oxygen Editor.

Here are a few samples of my work (where I am the sole writer unless otherwise noted):

External data services (for application developers)

The External data services documentation provides conceptual, task, and reference information to help users understand why they might want to implement an external data service solution, and basic guidance on how to set one up. This overview information and data flow diagram make it easy for users to understand what properties in the product can use the external data service, and how the data request is submitted to the external repository and how the data appears in the user interface.

I drafted the original diagram and conceptual information based on information that I received verbally from meetings and presentations.

Sync services (for end users)

Sync was a highly anticipated new feature in IBM Content Navigator Version 2.0.3. I was responsible for doing the information architecture and writing for the sync documentation, including the following topics:
  • Syncing content across multiple devices
  • Scenario: Syncing documents and folders that are shared in a teamspace
  • Tips for syncing items in IBM Content Navigator

Constructing a URL for IBM Content Navigator

Developers can learn how to construct URLs for IBM Content Navigator, which has many potential uses in application development. I included examples for each URL parameter and also sought after and included a sample scenario to help illustrate how these URLs can be integrated in business solutions.

An alphabetical list of stages

The Alphabetical list of stages is a reference document that lists all of the stages provided in InfoSphere DataStage and QualityStage. This document makes it easy for users to understand how each stage is obtained, where each stage is located in the product palette, and where they can go in the documentation for more information about each particular stage. Previously, this information could not be found in any one documentation resource.

To create a complete draft, I gained access to a working build and used the product to find the palette location and the job type information for each stage.

A "first-use" guide

This Getting started with Cubing Services guide was a project that I worked on sometime during my internship at IBM Silicon Valley Labs, San Jose, Calif (2009). External users needed an easy-to-use, complete guide that could quickly get them started with the product. Previously, the information for getting started was scattered among several different places in the information center and users had trouble finding the right information to get up and running quickly.

This guide needed to be especially easy to use, understand, and complete quickly. Customer calls on this issue grew costly and it was determined that we needed a more efficient, thorough guide that contained only the most basic, essential user tasks.

I partnered with my technical editor and developer to turn the various pieces into one clear, self-contained "getting started" guide. We ended up completing the guide with praise and flying colors. Soon after the project was complete, the developer I had been working with sent it to a team member who was already seeking the type of information covered by the guide. After he sent the documentation, the colleague responded with satisfaction for the "well-written" guide that was both clear and useful.
