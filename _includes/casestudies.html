{%- assign casestudies = site.data.casestudies -%} > Here are some stories from my time as a producer. ## Case Study: User Testing I was asked to work with game directors to pick our next map for postlaunch production, based on user testing feedback. - The director said the data wasn't useful because you couldn't tell who was giving feedback (important context about the tester's playstyle and experienc level), so you'd see the feedback grouped together, like: "too big" "just right" "snipers" "close combat"--all in one anonymized block. So I asked to see the feedback, tracked down where the data came from, and asked for the raw data. - The raw data wasn't enough (it gave a list of maps tested per day, but no information about which order the maps were tested, which made comments like "I liked the first one better" actually useless. To re-represent the data, I asked for a key of which participants played which maps, in which order, on which days. - I then used some Excel magic to lay out all the feedback comments with the context of which maps they were actually talking about. (I gave this information and feedback to user testing so they can improve if they wished to.) - The thing that makes me feel proud about this is that this was the state of user testing for years, and I was the first person to (1) ask what useless meant, and (2) figure out what was happening and solve the problem. Hearing that this was a pain, I investigated why we don't build all the executables at build (e.g., the tradeoffs for how long it would take versus how much it costs devs to have to rebuild each time). - Then I learned that they were all already being built! I can't remember if I discovered this on my own by checking the timestamp on those executables on my local machine, of if I asked an engineer. - Upon realizing this, I was able to update all the documentation and do deskside evangelization in the kindest and most discreet way possible (some devs had not realized this for years and I didn't want them to grieve those hours wasted over the years too badly). This type of discovery is why I love watching users work in real time, and why I'll always ask for opportunities to do this type of shadowing and get the chance to build real relationships with the people I'm supposed to be serving. ---
